2. API client (src/api/client.js)
A fetch wrapper that:

Prefixes every request with http://localhost:8000
Reads access_token from localStorage and adds Authorization: Bearer <token> header on every request
On 401 response: clears localStorage and redirects to /login
On any non-ok response: throws an error with the response JSON detail message
Exports a typed apiClient with get, post, patch, delete methods


3. Auth context (src/context/AuthContext.jsx)
A React context that:

On mount calls GET /users/me to get the current user
Stores { user, isLoading, isAuthenticated } in context
Exposes login(email, password) — calls POST /auth/login, stores token, fetches user
Exposes logout() — clears token, redirects to /login
Exposes these role helpers derived from user.roles:
jsisCOD
isSalesManager
isSalesSpecialist
isProcurementManager
isProcurementSpecialist
hasSalesAccess        // Sales Specialist OR Sales Manager OR COD
hasProcurementAccess  // Procurement Manager OR Procurement Specialist OR COD

user.roles is an array of objects: [{ id, name, description }] — check by roles.some(r => r.name === "COD")


4. Login page (/login)

No sidebar — full page centered card on bg-gray-50
"Commerce-Flow" title above the card
Fields: Email, Password
"Sign in" button — full width, bg-gray-900 text-white
On submit: calls login() from AuthContext → on success redirects to /requests
On error: shows "Invalid email or password" inline below the form
If already authenticated redirect away from login automatically


5. Protected route wrapper
A <ProtectedRoute> component that:

Shows a loading spinner while isLoading is true
Redirects to /login if not authenticated
Optionally accepts allowedRoles array — redirects to /unauthorized if user's roles don't match
